Git large file versioning

Nov 22, 2021
1 minute read
  1. Download git-lfs on https://git-lfs.github.com/, and run the installation package for windows.
  2. Once downloaded and installed, set up Git LFS for your user account by running:git lfs install. You only need to run this once per user account.
  3. In each Git repository where you want to use Git LFS, select the file types you’d like Git LFS to manage (or directly edit your .gitattributes). You can configure additional file extensions at anytime.git lfs track "*.psd"
  4. Now make sure .gitattributes is tracked:git add .gitattributes
  5. Note that defining the file types Git LFS should track will not, by itself, convert any pre-existing files to Git LFS, such as files on other branches or in your prior commit history. To do that, use the git lfs migrate[1] command, which has a range of options designed to suit various potential use cases.
  6. There is no step three. Just commit and push to GitHub as you normally would; for instance, if your current branch is named main: 
    git add file.psd
git commit -m "Add design file"
git push origin main
git github
